# Graphify Knowledge Graph Skill

You have access to the full knowledge graph of this codebase, generated by Graphify.
The graph is at `graphify-out/graph.json` and includes extracted AST relationships plus
inferred semantic edges.

## Available Operations

### Query the graph
```
graphify query "<natural-language-question>" --graph graphify-out/graph.json
```
This does BFS/DFS traversal to answer architecture questions.

### Find architectural hubs (god nodes)
```
graphify god-nodes --top 15 --graph graphify-out/graph.json
```

### Shortest path between two nodes
```
graphify path "NodeA" "NodeB" --graph graphify-out/graph.json
```

### Explain a node and its neighbors
```
graphify explain "ClassName" --graph graphify-out/graph.json
```

### Reverse impact analysis
```
graphify affected "ClassName" --depth 3 --graph graphify-out/graph.json
```

### Generate a collapsible tree HTML
```
graphify tree --graph graphify-out/graph.json
```

## Regenerating the Graph
If code has changed, re-extract:
```
graphify extract . --code-only --force
```

## Integration Command
To install graphify as a skill in external platforms:
```
graphify install --platform claude   # Claude Code
graphify install --platform cursor   # Cursor IDE
graphify install --platform windsurf # Windsurf
```

## Usage Tips
- Use `god-nodes` first to understand architectural hubs
- Use `path A B` to trace how two components are connected
- Use `affected X` before refactoring — find all code that depends on X
- Use `query "how does X relate to Y?"` for open-ended exploration
- The GRAPH_REPORT.md contains community analysis showing module cohesion
